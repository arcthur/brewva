the gateway daemon, credentials/inspect/insights helpers, and channel hosts.

## Mode Selection

Primary modes:

- interactive mode by default
- one-shot text mode with `--print`
- one-shot JSON mode with `--mode json` or `--json`
- undo, redo, and replay modes
- scheduler daemon mode
- channel gateway mode

Mode resolution is fail-fast. Incompatible input surfaces return a parse error
instead of reviving retired prompt-loop behavior.

## Input Contract

- `--task` and `--task-file` are mutually exclusive.
- Prompt text overrides `TaskSpec.goal` when both are provided.
- Non-TTY stdin/stdout defaults to text print mode unless another mode is
  explicit.
- `--replay`, `--undo`, and `--redo` are mutually exclusive and cannot combine
  with task inputs.
- `--help` and `--version` return exit code `0`; parse and validation failures
  return exit code `1`.

## Authority Boundary

CLI flags and slash commands are operator veneers. They select host behavior or
extension entrypoints; they do not create new kernel authority. Durable
receipt and event semantics live under `docs/reference/runtime.md` and
`docs/reference/events/README.md`.
