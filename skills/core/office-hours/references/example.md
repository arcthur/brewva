# Concrete Example

Input: "I want to build an AI app that helps engineers learn any codebase."

Office-hours first classifies this as startup mode unless the user says it is a
fun side project. It asks for the specific engineer, current workaround, and
moment of urgency before proposing a product surface. A weak answer produces an
assignment like "watch two onboarding sessions and write down the first five
commands they run." A stronger answer can produce options such as a read-only
first-day briefing, an onboarding task coach, and a code-tour artifact that can
be shared in a PR.
