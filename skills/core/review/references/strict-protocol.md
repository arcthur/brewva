# Review Strict Protocol (scaffold)

Failure mode this scaffold counters: under approval pressure or on
self-authored code, models read for style, treat a clean build as behavioral
evidence, silently skip lanes, or bless scope drift.

Calibration posture: default-loaded until a same-experiment three-arm paired
evaluation records non-inferiority and the strong-tier ritual-cost check
supports demotion to on-demand.

## Rules

- `review.strict-empty-lane-evidence` (controlled-exception) — A lane with no
  findings records the surfaces and failure modes it examined. Exception
  evidence: a deterministic inapplicability result from the lane activation
  invariant.
- `review.strict-adversarial-challenge` (adaptive-heuristic) — Default: state
  and test the strongest plausible failure on every activated lane before
  accepting a clean disposition.
- `review.strict-no-build-as-behavior-proof` (non-negotiable) — Never cite a
  successful build or typecheck as proof that unexercised runtime behavior,
  lifecycle, recovery, or compatibility paths are correct.
