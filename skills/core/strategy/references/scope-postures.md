# Strategy Review Scope Postures

Use this reference when `strategy` must choose how aggressively to shape
the next plan.

## The Three Postures

- expand: the current wedge is too timid relative to the leverage available now
- hold: the current wedge is already well sized and should move straight into
  design
- narrow: the current proposal hides the real learning loop behind too much
  scope

## What To Look For

Prefer `expand` when:

- a slightly broader slice unlocks a clearly better user outcome
- the extra surface is tightly adjacent, not a roadmap tangent

Prefer `hold` when:

- the current wedge is coherent, bounded, and worth shipping as-is
- more scope would mostly add delay or risk without clearer learning

Prefer `narrow` when:

- the plan touches too many surfaces before proving the core bet
- the real user value can be tested with a smaller slice

## Strategic Interrogation

Before choosing a posture, ask:

- why this wedge matters now instead of one cycle later
- what becomes true for the user if this ships on time
- what must remain explicitly deferred even if the wedge expands
- whether the proposal is hiding uncertainty behind a larger feature list

## Anti-Patterns

- ambition theater: expanding because the bigger idea sounds impressive
- fear shrinkage: refusing to cut scope because it feels less visionary
- false neutrality: describing multiple postures but refusing to recommend one
